STEP 2: FALLBACKS (bash-only — do not fish-ify)

  Requires /bin/bash (uses arrays, ${@:3}, ${args[-1]}).

  ```bash
  command -v rg  >/dev/null 2>&1 || rg()  { grep -P --color=never "$@"; }
  command -v bat >/dev/null 2>&1 || bat() {
    local args=() range=""
    while [[ $# -gt 0 ]]; do
      case "$1" in --plain) shift ;; -r) shift; range="$1"; shift ;; *) args+=("$1"); shift ;; esac
    done
    if [[ -n "$range" ]]; then sed -n "${range%%:*},${range##*:}p" "${args[-1]}"; else cat "${args[@]}"; fi
  }
  command -v fd  >/dev/null 2>&1 || fd() { local pat="$1"; shift; find "$@" -name "$pat" 2>/dev/null; }
  command -v sd  >/dev/null 2>&1 || sd() { sed -i "s/$1/$2/g" "${@:3}"; }
  export -f rg bat fd sd 2>/dev/null; true
  ```

 EXECUTION

RULES:
  Work continuously. Batch tool calls. Parallel when independent.
  >60 lines output → file first. >3 calls with no findings → flush.
  Blocked → DEFERRED, continue. Save after EVERY pass. Max 3 retries.
  Never re-read same region twice. >10 items → sub-chunks of 5.
  Write findings: echo "F-N ..." >> /home/claude/audit-findings.txt

TOOL MANDATES:
  RUST FIRST    rg over grep, fd over find, bat over cat/less, sd over
                sed -i. Fall back only if unavailable.
  PARALLEL      Every independent rg/fd call backgrounded with & + wait.
                Group related checks into ( ... ) & subshells. No serial
                rg loops.
  BATCH         Compound commands in single tool calls. Multiple rg in
                one subshell. One bash_tool call per pass, not per check.
  FISH SHELL    Use fish -c '...' for function body extraction with
                depth-tracking (handles nested functions correctly).
                Use rg -A for non-function context extraction only
                (e.g., nearby comments, short fixed blocks). NEVER use
                rg -A for function bodies — same nesting bug as sed -n.
  PCRE2 GUARD   rg -oP/-cP requires PCRE2. If unavailable, fall back
                to rg -o + grep -oP, or restructure the pattern.
                rg -Un (multiline) needs rg ≥11; fall back to awk for
                adjacent-line patterns if rg multiline unavailable.
  OUTPUT CAPS   Every rg command producing multi-line output MUST pipe
                through | head -N. Use: head -30 for high-hit patterns
                (set, sudo, command), head -50 for medium, head -20 for
                low-hit. Exceptions: rg -c (count-only) needs no cap.

CONTEXT BUDGET:
  Total tool output in context must not exceed 3000 lines across
  ALL passes. Budget per pass:
    PASS 1 (gather+analyze):  800 lines tool output max
    PASS 2 (runtime):         600 lines tool output max
    PASS 3 (finalize):        200 lines tool output max
    Overhead (analysis+findings+reasoning): 1400 lines reserve
    NOTE: Reasoning competes with this budget. If analysis
    is complex, reduce data collection caps proportionally.
    GUIDELINE: If PASS 1 findings exceed 30, cap PASS 2 to
    400 lines and reduce PASS 1 to 600 lines on retry.
    Freed 400 lines shift to overhead reserve (1800 total).

  RULES:
  - Raw data → disk only. Never paste >100 lines of rg output
    into context.
  - After each data collection step: wc -l all output files,
    report line counts only.
  - Findings → /home/claude/audit-findings.txt (append).
  - Summary → context (counts + severity + check numbers only).
  - If a single tool call returns >200 lines: truncate, note
    "[truncated, full output in $D/filename.txt]".

STALL PREVENTION:
  If approaching context limit (>20 tool calls or >25 messages):

  2. Skip remaining phases, mark as SKIPPED
  3. Jump to PASS 3 FINALIZE
  4. Produce partial spec with COVERAGE line

  MED (correctness) | LOW (style) | INFO (observation)

  Per finding: F-N | Severity | Phase | Checks | Lines | Category | Detail | Fix

  ```
  FIXES SPEC: <script> <old-ver> → <new-ver>
  Date: YYYY-MM-DD | Findings: N (N CRIT, N HIGH, N MED, N LOW, N INFO)

  --- FIX INSTRUCTIONS ---
  F-1  [SEV]  <summary>
       Checks: N,N  Lines: N,N  Category: <checklist item>
       Before: <exact text>
       After:  <exact replacement>
       Verify: <command>

  --- INFO (no action) ---
  I-1  [INFO]  <desc>  Checks: N  Lines: N

  --- VERSION ---
  Bump: <old> → <new>
  Changelog: v<new> <summary>.

  --- VERIFICATION ---
  1. fish --no-execute script.fish
  2. rg '<old-version>' script.fish README.md  (expect 0)
  3. rg '<new-version>' script.fish README.md  (expect N)
  ```

  Before/After must be exact and unique within the file (required for
  str_replace). If not unique, include enough surrounding context to
  disambiguate. Spec self-contained. Version bump if any actionable
  fixes. INFO listed separately.

  UNIQUENESS VERIFICATION (run DURING and AFTER spec generation):
  Before adding each Before/After pair, verify uniqueness first:
  ```bash
  verify_unique() {
    local pat="$1"
    local c=$(rg -cF -- "$pat" /home/claude/audit-src/script.fish 2>/dev/null || echo 0)
    if [ "$c" != "1" ]; then
      echo "NON-UNIQUE ($c hits): $pat -- expand context"
      return 1
    fi
  }
  ```
  If non-unique (0 or 2+ hits), expand Before/After with surrounding
  lines until unique. Post-write full check:
  ```bash
  rg -o 'Before: .*' fixes-spec-*.txt | while IFS= read -r line; do
    pat="${line#*Before: }"
    c=$(rg -cF -- "$pat" /home/claude/audit-src/script.fish 2>/dev/null || echo 0)
    [ "$c" != "1" ] && echo "NON-UNIQUE ($c hits): $pat"
  done
  ```

STEP 4: PACKAGE

  ```bash
  S=script; V=x.y.z; OUT=/mnt/user-data/outputs
  cp /home/claude/audit-${S}-${V}.txt /home/claude/fixes-spec-${V}.txt "$OUT/"
  cd /home/claude && zip "${OUT}/audit-${S}-${V}.zip" \
    audit-${S}-${V}.txt fixes-spec-${V}.txt
  ```

DELIVERABLES: audit-<script>-<version>.txt + fixes-spec-<version>.txt (zipped)
